Cechy modyfikowania formatowania w:
HTML CSS
W HTML, modyfikacja wyglądu elementów strony jest bardzo żmudna, ponieważ atrybuty i znaczniki odpowiedzialne za formatowanie są porozrzucane w różnych miejscach kodu, mieszając się ze strukturą dokumentu. W CSS następuje rozdzielenie struktury i prezentacji dokumentów. Wszystkie polecenia dotyczące formatowania można umieścić w jednym pliku (tzw. arkuszu stylów) i powiązać je z odpowiednimi elementami w HTML. Taki sposób sprawia, że modyfikacja wyglądu strony jest znacznie sprawniejsza.

Nowe możliwości CSS

  1. Przestarzałe atrybuty i znaczniki w HTML, które odpowiadają za formatowanie, stopniowo będą wycofywane na rzecz deklaracji CSS, rekomendowanych przez producentów przeglądarek internetowych.
  2. Dokumenty używające arkuszy stylów są zazwyczaj bardziej przejrzyste i krótsze.
  3. CSS pozwala na łatwe zarządzanie całymi seriami dokumentów za pomocą zewnętrznych arkuszy stylów.
  4. Za pomocą klas selektorów oszczędzamy na pisaniu kodu, ponieważ w jednym miejscu możemy określić wszystkie atrybuty formatowania odnoszące się do wielu elementów, które mają wyglądać identycznie. Dzięki temu modyfikacje są szybsze, ponieważ zmiany wprowadzamy tylko w jednym miejscu.
  5. CSS umożliwia stosowanie różnorodnych jednostek oraz metod definiowania kolorów.
  6. CSS oferuje różne wartości pogrubienia czcionki (aż 9 rodzajów).
  7. CSS daje dodatkowe możliwości formatowania tekstu, takie jak zaawansowane style.
  8. Efektowne filtry graficzne dostępne w CSS umożliwiają dodanie ciekawych efektów wizualnych.
  9. CSS pozwala na określenie wyglądu dokumentów podczas ich drukowania, co daje dużą elastyczność.

Wady CSS
  1. CSS ma skomplikowaną specyfikację, której składnia jest wrażliwa na błędy, co może prowadzić do problemów.
  2. Brak sygnalizacji błędów w kodzie CSS podczas jego wykonywania utrudnia identyfikację problemów.
  3. Różne przeglądarki mogą błędnie obsługiwać niektóre właściwości CSS, co prowadzi do problemów z kompatybilnością.
